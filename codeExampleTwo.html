SQL Query

The Query example is to find one genre of movie where they are rated from highest to lowest.

Getting the correct table

First of all I used the "SELECT" keyword to select the columns I wanted. AS was used to rename the columns. I then used the keyword "FROM" to get the columns from the tables I needed. I used the "FULL JOIN", selected the table and linked them together with the primary key "mov_id". I then used "WHERE" so I could create a sub query and link through the movie genres table to get to the genres table. This is where I was able to specify the genres I wanted to see.

The code I have written will show all the movies that are rated in the "Drama" genre. Three of which returns NULL. Unfortunely the tables didnt have to many stored rated movies for each genre. I would of entered more but I do not have the authorization. To do this I would of used "INSERT INTO" selected the table and columns then used "VALUES" and inputed the values. Finally I used "ORDERED BY" to order the results but rating in ascending order and then title by ascending order.

Results

There is only one result that is rated. The other three are "NULL". If I just wanted rated movies I would have inserted "rev_stars IS NOT NULL" inbetween the first "WHERE" and "M.mov_id" then just added an "AND" to add the sub query. If there were more results and I wanted to change the query to find the highest rated movie in its genre. I would of either used a "LIMIT" in the correct order to get the highest rated movie in its genre or created another subquery and used the "MAX" aggregate function to select the highest rated movie in its genre.
